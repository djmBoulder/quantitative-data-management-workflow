# 10 Capstone Checklist

Use this checklist before submitting your final data management portfolio.

## Pathway Use

- [ ] I identified my portfolio pathway: Stata, R, or dual Stata/R.
- [ ] I understand that one primary workflow is sufficient unless my instructor requires both.
- [ ] Optional dual-pathway extension: I can run both scripts and show that they implement the same major data decisions.

## Reproducible Script

- [ ] I submitted one complete primary-pathway Stata do-file or R script.
- [ ] My script runs from the project root.
- [ ] My script uses relative paths.
- [ ] My script creates needed output folders.
- [ ] My script starts a log or text output record.
- [ ] My script can recreate the final outputs without manual steps.

## Raw Data Protection

- [ ] I used only synthetic course data.
- [ ] I did not manually edit raw CSV files.
- [ ] I did not use private, student, restricted, licensed, or confidential data.
- [ ] I documented the source files used in the workflow memo.

## Cleaning And Missing Data

- [ ] I standardized variable names.
- [ ] I cleaned and recoded selected variables.
- [ ] I preserved the meaning of source variables in derived variables.
- [ ] I converted special missing codes to system missing values.
- [ ] I checked missingness after recoding.
- [ ] I documented missing-data decisions.

## Appends And Merges

- [ ] I appended the two person-level files.
- [ ] I checked row counts before and after append.
- [ ] I checked duplicate respondent-year keys.
- [ ] I imported the region-level contextual file.
- [ ] I checked the region merge key.
- [ ] I merged region context onto person-level data.
- [ ] I checked matched and unmatched merge cases.
- [ ] I documented any unresolved merge issues.

## QA Evidence

- [ ] I checked row counts.
- [ ] I checked variable names or required variables.
- [ ] I checked numeric ranges.
- [ ] I checked valid categories.
- [ ] I checked duplicate keys.
- [ ] I checked sample restrictions.
- [ ] I included a QA checklist or QA report.

## Final Outputs

- [ ] I created an analysis-ready dataset.
- [ ] I created a data dictionary or codebook.
- [ ] I created a workflow memo.
- [ ] I included evidence that outputs were regenerated from the script.
- [ ] I kept generated files in appropriate ignored folders unless instructed otherwise.

## Workflow Memo

- [ ] My memo identifies raw inputs.
- [ ] My memo identifies the script and log.
- [ ] My memo lists final outputs.
- [ ] My memo explains major cleaning decisions.
- [ ] My memo explains missing-data handling.
- [ ] My memo explains append and merge checks.
- [ ] My memo explains sample restrictions.
- [ ] My memo notes known limitations.
