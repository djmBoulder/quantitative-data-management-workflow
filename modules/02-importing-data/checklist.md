# 02 Importing and Inspecting Data Checklist

Use this checklist before submitting the module assignment.

## Pathway Use

Complete the required lab checks for your primary pathway. Use the other software section as conceptual review unless you are completing the optional dual pathway or your instructor requires both.

## File Types

- [ ] I can explain what a CSV file is.
- [ ] I can explain what an Excel `.xlsx` file is.
- [ ] I can explain what a Stata `.dta` file is.
- [ ] I can explain why fixed-width/plain-text files need layout documentation.

## Workflow Setup

- [ ] I ran my primary-pathway lab script from the repository root.
- [ ] I used relative paths rather than absolute paths from my own computer.
- [ ] I did not manually edit the raw CSV or Excel file.
- [ ] I checked the data dictionary before interpreting variables.

## Stata Lab

- [ ] Stata pathway or dual pathway: I ran `modules/02-importing-data/stata_lab.do`.
- [ ] My Stata workflow imported the CSV file.
- [ ] My Stata workflow imported the Excel file.
- [ ] My Stata workflow inspected names, types, missingness, and selected values.
- [ ] My Stata workflow standardized variable names.
- [ ] My Stata workflow saved a working `.dta` copy without changing raw files.

## R Lab

- [ ] R pathway or dual pathway: I ran `modules/02-importing-data/r_lab.R`.
- [ ] My R workflow read the CSV file.
- [ ] My R workflow read the Excel file.
- [ ] My R workflow inspected names, types, missingness, and selected values.
- [ ] My R workflow standardized variable names.
- [ ] My R workflow saved working `.rds` and `.csv` copies without changing raw files.

## Import Inspection Memo

- [ ] I identified the raw files used in this module.
- [ ] I described variable-name problems visible after import.
- [ ] I listed variable-name changes made during the lab.
- [ ] I described type problems or type surprises.
- [ ] I identified missing-value or special-code issues.
- [ ] I named at least three import problems for later modules to clean or document.
- [ ] I explained how the workflow protects raw files.

## Ready for the Next Module

- [ ] I understand that importing is not the same as cleaning.
- [ ] I can find the working files created by the labs.
- [ ] I can find the log and inspection files created by the labs.
- [ ] I am ready to use the imported working data for documentation and codebook work.
